    62→### ⚠️ Step 5 BUILD — MANDATORY TOC Gate
    63→
    64→**Before writing ANY ReportLab code, answer this question:**
    65→
    66→> **Does this document have a Table of Contents?**
    67→
    68→| Answer | DocTemplate | Build Method | TOC Implementation |
    69→|--------|-------------|--------------|--------------------|
    70→| **YES** | `TocDocTemplate` | `doc.multiBuild(story)` | `TableOfContents()` + heading `bookmark_name/level/text/key` attributes |
    71→| **NO** | `SimpleDocTemplate` | `doc.build(story)` | N/A |
    72→
    73→**⚠️ If you answered YES, these are ALL mandatory — missing any one = broken TOC:**
    74→
    75→```python
    76→# 1. MUST define TocDocTemplate (NOT SimpleDocTemplate)
    77→class TocDocTemplate(SimpleDocTemplate):
    78→    def afterFlowable(self, flowable):
    79→        if hasattr(flowable, 'bookmark_name'):
    80→            level = getattr(flowable, 'bookmark_level', 0)
    81→            text = getattr(flowable, 'bookmark_text', '')
    82→            key = getattr(flowable, 'bookmark_key', '')
    83→            self.notify('TOCEntry', (level, text, self.page, key))
    84→
    85→# 2. MUST add TableOfContents to story
    86→toc = TableOfContents()
    87→toc.levelStyles = [toc_level0, toc_level1]  # define your styles
    88→story.append(toc)
    89→story.append(PageBreak())
    90→
    91→# 3. MUST set bookmark attributes on every heading
    92→def add_heading(text, style, level=0):
    93→    key = f'h_{hashlib.md5(text.encode()).hexdigest()[:8]}'
    94→    p = Paragraph(f'<a name="{key}"/>{text}', style)
    95→    p.bookmark_name = key
    96→    p.bookmark_level = level
    97→    p.bookmark_text = text
    98→    p.bookmark_key = key
    99→    return p
   100→
   101→# 4. MUST use multiBuild (NOT build)
   102→doc = TocDocTemplate(output_path, pagesize=A4, ...)
   103→doc.multiBuild(story)
   104→```
   105→
   106→**🚫 FORBIDDEN patterns (these produce broken/fake TOC):**
   107→```python
   108→# ❌ Hard-coded TOC items with manual page numbers
   109→toc_items = [('Chapter 1', '3'), ('Chapter 2', '5')]  # WRONG - not clickable, page numbers will be wrong
   110→
   111→# ❌ SimpleDocTemplate + build() with TOC
   112→doc = SimpleDocTemplate(...)  # WRONG if you have TOC
   113→doc.build(story)              # WRONG - TOC entries won't populate
   114→
   115→# ❌ TableOfContents without TocDocTemplate
   116→toc = TableOfContents()
   117→story.append(toc)
   118→doc = SimpleDocTemplate(...)  # WRONG - no afterFlowable to feed TOC
   119→```

   127→### 2a. Generate Palette (MANDATORY)
   128→
   129→**Run this command FIRST. Copy-paste its output directly into your Python script:**
   130→
   131→```bash
   132→python3 "$PDF_SKILL_DIR/scripts/pdf.py" palette.cascade --title "<document title>" --mode minimal --format reportlab

   216→## Step 3: EDIT - Content Transformation
   217→
   218→**Before writing any code, restructure raw text into visual roles.** This is the most impactful quality step - skipping it produces "Word document with a PDF extension".
   219→
   220→### 3a. Typographic Role Extraction
   221→
   222→Parse raw text and categorize content into visual roles:
   223→
   224→| Role | ReportLab Mapping | Description |
   225→|------|------------------|-------------|
   226→| **Hero / Display** | Cover title (36-42pt) or section opener | 1-5 words, the emotional hook |
   227→| **Kicker / Eyebrow** | Subtitle or section intro (10-12pt, muted) | Tiny context line above main heading |
   228→| **Data Sculpture** | CalloutBox or bold stat block | Extract impactful numbers (97%, $4.2M, -45ms) |
   229→| **Pull Quote** | BlockQuote (italic + left indent 24pt + accent border) | Most provocative sentence, standalone |
   230→| **Body** | Standard paragraph | Everything else |
   231→
   232→**Rule**: Before writing any body paragraph, scan for numbers with units. Every metric like "revenue grew by 12%" or "latency dropped to 45ms" MUST be extracted into a CalloutBox or metrics table - never buried in paragraph prose. See §Data-to-Ink Ratio Rules below.
   233→
   234→### 3b. Section Pacing
   235→
   236→- If any H1 section has **>400 words** of continuous body text without visual breaks, split into H2 subsections or insert a visual break (table, chart, callout)
   237→- Aim for at least 1 visual element (table/chart/callout) per 2-3 pages of body text
   238→- Dense text walls are the #1 sign of poor report design
   239→
   240→### 3c. Content Sanitization for External Sources (MANDATORY when source is PDF/OCR/web)
   241→
   242→When content is extracted from an external PDF, OCR output, or web scrape, it may contain invisible garbage characters that cause garbled rendering. **You MUST sanitize the text content before writing it into any `Paragraph()` or `canvas.drawString()`.**
   243→
   244→This is different from `code.sanitize` (which cleans Python source code). Content sanitization cleans the **text data itself**.
   245→
   246→```python
   247→import re
   248→
   249→def content_sanitize(text: str) -> str:
   250→    """Strip invisible/control chars from externally-sourced text."""
   251→    # Remove control chars (except \n \t)
   252→    text = re.sub(r'[\x00-\x08\x0b\x0c\x0e-\x1f\x7f]', '', text)
   253→    # Remove zero-width chars
   254→    text = re.sub(r'[\u200b-\u200f\u2028-\u202f\u2060\ufeff]', '', text)
   255→    # Remove replacement char (upstream encoding error)
   256→    text = text.replace('\ufffd', '')
   257→    # Remove variation selectors
   258→    text = re.sub(r'[\ufe00-\ufe0f]', '', text)
   259→    # Remove private use area chars
   260→    text = re.sub(r'[\ue000-\uf8ff]', '', text)
   261→    return text
   262→```
   263→
   264→**When to use:**
   265→| Source | Must sanitize? |
   266→|--------|---------------|
   267→| User typed text in prompt | No (usually clean) |
   268→| Extracted from PDF via `pdf.py extract.text` | **YES** |
   269→| OCR output | **YES** |
   270→| Web scrape / HTML strip | **YES** |
   271→| LLM-generated content | No (usually clean) |
   272→
   273→⚠️ **Do NOT confuse** `code.sanitize` (Step 6, cleans Python code) with `content_sanitize()` (Step 3, cleans text data). They solve different problems.

   302→## Character Safety Rule (MANDATORY)
   303→
   304→Three rules for safe character handling in ReportLab PDFs:
   305→
   306→**a) Superscripts and subscripts**: Use `<super>`/`<sub>` tags, never raw Unicode superscript/subscript characters (e.g., `\u00b2`, `\u2082`).
   307→
   308→**b) Emoji**: ReportLab cannot render emoji. If content contains emoji, use Creative brief (HTML + Playwright).
   309→
   310→**c) Font fallback for mixed CJK/Latin text (MANDATORY)**: After registering fonts, you **MUST** call `install_font_fallback()` once. This automatically wraps missing-glyph characters in `<font>` tags inside every `Paragraph()`. No manual `<font name="...">` wrapping needed for mixed Chinese-English text. **Skipping this call is the #1 cause of Helvetica garbled text in CJK documents.**
   311→
   312→**d) canvas.drawString / drawRightString CJK font rule (MANDATORY)**: `install_font_fallback()` only works on `Paragraph()` objects. It does NOT affect `canvas.drawString()` / `canvas.drawRightString()` / `canvas.drawCentredString()`. If the text passed to any `canvas.draw*String()` call contains CJK characters (Chinese/Japanese/Korean), you **MUST** call `canvas.setFont('NotoSerifSC', size)` (or another registered CJK font) before drawing. Using FreeSerif / Helvetica / Times for CJK canvas text = garbled output.
   313→
   314→```python
   315→# ❌ WRONG — FreeSerif has no CJK glyphs, '科大讯飞' will be garbled
   316→canvas.setFont('FreeSerif-Italic', 8)
   317→canvas.drawRightString(x, y, 'SZSE: 002230  ·  科大讯飞')
   318→
   319→# ✅ CORRECT — use CJK font for mixed text
   320→canvas.setFont('NotoSerifSC', 8)
   321→canvas.drawRightString(x, y, 'SZSE: 002230  ·  科大讯飞')
   322→
   323→# ✅ ALSO CORRECT — pure English text can use FreeSerif
   324→canvas.setFont('FreeSerif-Italic', 8)
   325→canvas.drawString(x, y, 'Prepared by Strategy Advisory Team')
   326→```
   327→
   328→Mathematical/relational operators (×, ÷, ±, ≤, √, ∑, ≅, ∫, π, ∠, Δ, etc.) are safe to use as literal characters in `Paragraph()` - both Noto Sans SC and FreeSerif have these glyphs.
   329→
   330→| Need | Correct Method | Correct Example |
   331→|------|---------------|---------|
   332→| Superscript | `<super>` tag in `Paragraph()` | `Paragraph('10<super>2</super> × 10<super>3</super> = 10<super>5</super>', style)` |
   333→| Subscript | `<sub>` tag in `Paragraph()` | `Paragraph('H<sub>2</sub>O', style)` |
   334→| Bold | `<b>` tag in `Paragraph()` | `Paragraph('<b>Title</b>', style)` |
   335→| Math operators | Literal char in `Paragraph()` | `Paragraph('AB ⊥ AC, ∠A = 90°, and ΔABC ≅ ΔDCF', style)` |
   336→| Scientific notation | Combined tags in `Paragraph()` | `Paragraph('1.2 × 10<super>8</super> kg/m<super>3</super>', style)` |
   337→
   338→```python
   339→from reportlab.platypus import Paragraph
   340→from reportlab.lib.styles import ParagraphStyle
   341→from reportlab.lib.enums import TA_LEFT, TA_CENTER, TA_JUSTIFY
   342→
   343→body_style = ParagraphStyle(
   344→    name="ENBodyStyle",
   345→    fontName="FreeSerif",
   346→    fontSize=10.5,
   347→    leading=18,
   348→    alignment=TA_JUSTIFY,
   349→)
   350→
   351→# CJK Body (Chinese documents)
   352→cn_body_style = ParagraphStyle(
   353→    name="CNBodyStyle",
   354→    fontName="NotoSerifSC", fontSize=10.5, leading=18,
   355→    alignment=TA_LEFT,      # ← NEVER TA_JUSTIFY for CJK
   356→    wordWrap='CJK',         # ← MANDATORY for CJK
   357→    firstLineIndent=2*4.5*mm,
   358→)
   359→
   360→# Superscript: area unit
   361→Paragraph('Total area: 500 m<super>2</super>', body_style)
   362→
   363→# Subscript: chemical formula
   364→Paragraph('The reaction produces CO<sub>2</sub> and H<sub>2</sub>O', body_style)
   365→
   366→# Scientific notation
   367→Paragraph('Speed of light: 3.0 × 10<super>8</super> m/s', body_style)
   368→
   369→# Combined
   370→Paragraph('E<sub>k</sub> = mv<super>2</super>/2', body_style)
   371→
   372→# Bold heading
   373→Paragraph('<b>Chapter 1: Introduction</b>', header_style)
   374→
   375→# Math symbols
   376→Paragraph('When ∠ A = 90°, AB ⊥ AC and ΔABC ≅ ΔDEF', body_style)
   377→```
   378→
   379→**Pre-generation check - before writing ANY string, ask:**
   380→> "Does this string contain a character outside basic CJK or Mathematical/relational operators?"
   381→> If YES → it MUST be inside a `Paragraph()` with the appropriate tag.
   382→> If it is a superscript/subscript digit in raw unicode escape form → REPLACE with `<super>`/`<sub>` tag.
   383→
   384→**NEVER rely on post-generation scanning. Prevent at the point of writing.**
   385→
   386→**Encoding safety - before writing ANY content text:**
   387→> "Does this string contain Japanese kana (の, が, は etc.) or rare Unicode symbols?"
   388→> If YES → REPLACE with safe plain Chinese equivalents. Japanese kana (Hiragana U+3040-U+309F, Katakana U+30A0-U+30FF) frequently corrupt to U+FFFD (�) when code passes through LLM output, heredoc, or terminal encoding layers.
   389→> Common safe replacements: `活の真鲷`→`活缔真鲷`, `盐烤の鲭鱼`→`盐烤鲭鱼`, `烤の鸡串`→`炭烤鸡串`.
   390→> If the character is genuinely needed, verify it survives a full write→read round-trip with `open(file, encoding='utf-8')`.

   394→## Font Setup (Guaranteed Success Method)
   395→
   396→### CRITICAL: Allowed Fonts Only
   397→**You MUST ONLY use the following registered fonts. Using ANY other font is STRICTLY FORBIDDEN.**
   398→
   399→| Font Name | Usage | Path |
   400→|-----------|-------|------|
   401→| `NotoSerifSC` | **Chinese body text (primary)** |  `/usr/share/fonts/truetype/noto-serif-sc/NotoSerifSC-Regular.ttf` |
   402→| `NotoSerifSC-Bold` | **Chinese headings (primary)** |  `/usr/share/fonts/truetype/noto-serif-sc/NotoSerifSC-Bold.ttf` |
   403→| `Noto Sans SC` | Chinese fallback / sans-serif | `/usr/share/fonts/truetype/chinese/NotoSansSC-Regular.ttf` |
   404→| `Noto Sans SC Bold` | Chinese headings (sans-serif) | `/usr/share/fonts/truetype/chinese/NotoSansSC-Bold.ttf` |
   405→| `SarasaMonoSC` | Chinese code blocks | `/usr/share/fonts/truetype/chinese/SarasaMonoSC-Regular.ttf` |
   406→| `FreeSerif` | English text, numbers, tables | `freefont/FreeSerif.ttf` |
   407→| `DejaVuSans` | Formulas, symbols, code | `/usr/share/fonts/truetype/dejavu/DejaVuSansMono.ttf` |
   408→
   409→**FORBIDDEN fonts (DO NOT USE):**
   410→- ❌ Arial, Arial-Bold, Arial-Italic
   411→- ❌ Helvetica, Helvetica-Bold, Helvetica-Oblique
   412→- ❌ Courier, Courier-Bold
   413→- ❌ Any font not listed in the table above
   414→
   415→> **Font file locations note:** On macOS, font paths may be under `~/.openclaw/workspace/fonts/` or `~/Library/Fonts/` instead of `/usr/share/fonts/`. The registration code below uses a single `FONT_DIR` variable — set it to the correct base path for your system. All font sub-paths (e.g. `truetype/noto-serif-sc/NotoSerifSC-Regular.ttf`) are appended to `FONT_DIR`.
   416→
   417→### Font Registration Template
   418→```python
   419→from reportlab.pdfbase import pdfmetrics
   420→from reportlab.pdfbase.ttfonts import TTFont
   421→from reportlab.pdfbase.pdfmetrics import registerFontFamily

   532→**Anti-tear (mandatory — but with height limit):**
   533→```python
   534→from reportlab.platypus import KeepTogether
   535→from reportlab.lib.pagesizes import A4
   536→from reportlab.lib.units import inch
   537→
   538→# Maximum height for KeepTogether blocks: 40% of page height
   539→# Blocks taller than this should NOT be kept together — they cause
   540→# the preceding page to be mostly empty when the block gets pushed down.
   541→MAX_KEEP_HEIGHT = A4[1] * 0.4  # ~336pt ≈ 12cm
   542→
   543→def safe_keep_together(elements):
   544→    """Wrap elements in KeepTogether only if their total height is reasonable.
   545→    If too tall, keep only the FIRST TWO elements together (e.g. title + table/image header)
   546→    to prevent title-content separation, while letting the rest paginate naturally.
   547→    """
   548→    total_h = 0
   549→    for el in elements:
   550→        w, h = el.wrap(A4[0] - 2*inch, A4[1])
   551→        total_h += h
   552→    if total_h <= MAX_KEEP_HEIGHT:
   553→        return [KeepTogether(elements)]
   554→    elif len(elements) >= 2:
   555→        # Keep at least title + first element together to prevent title orphaning
   556→        return [KeepTogether(elements[:2])] + list(elements[2:])
   557→    else:
   558→        return list(elements)
   559→
   560→# Heading stays with first paragraph
   561→story.extend(safe_keep_together([
   562→    heading_paragraph,
   563→    first_body_paragraph,
   564→]))
   565→
   566→# Image + caption: keep together ONLY if image is small
   567→story.extend(safe_keep_together([image, caption_paragraph]))
   568→
   569→# Table title + table: keep together ONLY if table is short
   570→if len(table_data) <= 15:
   571→    story.extend(safe_keep_together([table_title, table]))
   572→else:
   573→    # Long table: just keep title with the table start
   574→    story.extend(safe_keep_together([table_title]))
   575→    story.append(table)
   576→```
   577→
   578→**⚠️ KEY INSIGHT:** `KeepTogether` is the #1 cause of “empty page with chart on next page”.
   579→When a `KeepTogether` block is taller than the remaining space on the current page,
   580→ReportLab pushes the **entire block** to the next page, leaving the current page mostly empty.
   581→**Always use `safe_keep_together()` to cap the maximum block height.**
   582→
   583→**⚠️ Note:** Do NOT use `KeepWithNext` - it is unreliable in ReportLab 4.x.

   595→### Cover Page: HTML/Playwright Unified Cover System
   596→
   597→**⚠️ Report route covers are ALWAYS generated via HTML/Playwright**, using the same 5-template system defined in `typesetting/cover.md`. This ensures visual consistency across all routes (Report, Creative, Academic) and avoids the limitations of ReportLab for complex cover layouts.
   598→

   600→1. Generate **body PDF** via ReportLab (start with TOC or content - **no cover in story[]**)

   687→**Insertion script (single output PDF):**
   688→```python
   689→from pypdf import PdfReader, PdfWriter
   690→
   691→A4_W, A4_H = 595.28, 841.89  # A4 in points
   692→
   693→def normalize_page_to_a4(page):
   694→    """Scale a page to A4 if its dimensions don't match."""
   695→    box = page.mediabox
   696→    w, h = float(box.width), float(box.height)
   697→    if abs(w - A4_W) > 2 or abs(h - A4_H) > 2:
   698→        page.scale_to(A4_W, A4_H)
   699→    return page
   700→
   701→def insert_cover(cover_pdf, body_pdf, output_pdf):
   702→    """Insert cover as first page of body PDF → single output file."""
   703→    writer = PdfWriter()
   704→    # Cover as page 1
   705→    cover_page = PdfReader(cover_pdf).pages[0]
   706→    writer.add_page(normalize_page_to_a4(cover_page))
   707→    # Body pages follow
   708→    for page in PdfReader(body_pdf).pages:
   709→        writer.add_page(normalize_page_to_a4(page))
   710→    writer.add_metadata({'/Title': 'Report Title', '/Author': 'Z.ai', '/Creator': 'Z.ai'})
   711→    with open(output_pdf, 'wb') as f:
   712→        writer.write(f)
   713→```

You must call `normalize_page_to_a4()` to unify dimensions before insertion.

   904→### Table Width Management (⚠️ Prevent Table Overflow)
   905→
   906→**Most common table bug: columns overflow the right page margin because colWidths are hardcoded without checking available space.**
   907→
   908→**Iron Rule: The sum of all colWidths MUST NOT exceed `available_width`.**
   909→
   910→```python
   911→from reportlab.lib.pagesizes import A4
   912→from reportlab.lib.units import inch
   913→
   914→page_width = A4[0]  # 595.28pt
   915→left_margin = 1.0 * inch   # 72pt (adjust to your doc's actual margins)
   916→right_margin = 1.0 * inch  # 72pt
   917→available_width = page_width - left_margin - right_margin  # ≈ 451pt
   918→
   919→# ─── Method 1: Proportional widths (RECOMMENDED) ───
   920→# Define column ratios, auto-scale to fit available_width
   921→col_ratios = [0.25, 0.40, 0.20, 0.15]  # must sum to 1.0
   922→col_widths = [r * available_width for r in col_ratios]
   923→table = Table(data, colWidths=col_widths)
   924→
   925→# ─── Method 2: Fixed + flex columns ───
   926→# Some columns have fixed width (numbers, dates), rest fills remaining space
   927→fixed_cols = {0: 80, 3: 60}  # col 0 = 80pt, col 3 = 60pt
   928→fixed_total = sum(fixed_cols.values())
   929→flex_count = 4 - len(fixed_cols)  # total columns minus fixed
   930→flex_width = (available_width - fixed_total) / flex_count
   931→col_widths = []
   932→for i in range(4):
   933→    col_widths.append(fixed_cols.get(i, flex_width))
   934→table = Table(data, colWidths=col_widths)
   935→
   936→# ─── Method 3: Let ReportLab auto-calculate (simplest, less control) ───
   937→# Pass colWidths=None - ReportLab auto-sizes based on content
   938→# ⚠️ Risk: may still overflow if content is too wide
   939→table = Table(data)  # no colWidths
   940→```
   941→
   942→**Checklist before adding any table:**
   943→1. Calculate `available_width` from your actual page size and margins
   944→2. Sum all colWidths - must be ≤ `available_width`
   945→3. For CJK text: characters are wider than Latin - budget ~12pt per CJK char at 10pt font
   946→4. If a column has long text (descriptions, policies), use Paragraph() wrapping inside cells (not plain strings) - plain strings don't wrap and will overflow
   947→5. Test with the longest expected content in each column
   948→
   949→**Common mistake - plain strings don't wrap:**
   950→```python
   951→# ❌ Plain string: will overflow if text is long
   952→data = [['Policy Name', 'Very long description that keeps going and going']]
   953→# ✅ Paragraph: wraps within column width
   954→from reportlab.platypus import Paragraph
   955→from reportlab.lib.styles import getSampleStyleSheet
   956→styles = getSampleStyleSheet()
   957→cell_style = styles['Normal']
   958→data = [[Paragraph('Policy Name', cell_style),
   959→         Paragraph('Very long description that keeps going and going', cell_style)]]
   960→```
   961→
   962→```
   963→Do you need auto-TOC?
   964→├─ YES → Use TocDocTemplate + doc.multiBuild(story)
   965→└─ NO → Use SimpleDocTemplate + doc.build(story)
   966→```
   967→
   968→| Requirement | DocTemplate | Build Method |
   969→|-------------|-------------|--------------|
   970→| Multi-page with TOC | `TocDocTemplate` | `multiBuild()` |

  1008→## Auto-Generated Table of Contents
  1009→
  1010→### ❌ FORBIDDEN: Manual Table of Contents
  1011→**NEVER manually create TOC with hardcoded page numbers.**
  1012→
  1013→### ⚠️ MANDATORY: TOC requires a cover page
  1014→**Unless the user explicitly requests no cover, if a document has a Table of Contents, it MUST have a cover page.** Structure: Cover (page 1) → TOC (page 2) → Content (page 3+). Do not generate a TOC without a preceding cover page.
  1015→
  1016→### ✅ ALWAYS use auto-generated TOC:
  1017→
  1018→```python
  1019→from reportlab.platypus import SimpleDocTemplate, Paragraph, PageBreak, Spacer
  1020→from reportlab.platypus.tableofcontents import TableOfContents
  1021→from reportlab.lib.utils import simpleSplit
  1022→import hashlib
  1023→
  1024→class TocDocTemplate(SimpleDocTemplate):
  1025→    def afterFlowable(self, flowable):
  1026→        if hasattr(flowable, 'bookmark_name'):
  1027→            level = getattr(flowable, 'bookmark_level', 0)
  1028→            text = getattr(flowable, 'bookmark_text', '')
  1029→            key = getattr(flowable, 'bookmark_key', '')
  1030→            # MUST pass key as 4th element - without it, TOC entries won't have clickable links
  1031→            self.notify('TOCEntry', (level, text, self.page, key))
  1032→
  1033→doc = TocDocTemplate("document.pdf", pagesize=letter)
  1034→story = []
  1035→
  1036→toc = TableOfContents()
  1037→toc.levelStyles = [
  1038→    ParagraphStyle(name='TOCHeading1', fontSize=14, leftIndent=20, fontName='FreeSerif'),
  1039→    ParagraphStyle(name='TOCHeading2', fontSize=12, leftIndent=40, fontName='FreeSerif'),
  1040→]
  1041→story.append(Paragraph("<b>Table of Contents</b>", styles['Title']))
  1042→story.append(toc)
  1043→story.append(PageBreak())
  1044→
  1045→def add_heading(text, style, level=0):
  1046→    # Generate a unique bookmark key for this heading
  1047→    key = 'h_%s' % hashlib.md5(text.encode()).hexdigest()[:8]
  1048→    p = Paragraph('<a name="%s"/>%s' % (key, text), style)
  1049→    p.bookmark_name = text
  1050→    p.bookmark_level = level
  1051→    p.bookmark_text = text
  1052→    p.bookmark_key = key  # This key enables clickable TOC links
  1053→    return p
  1054→
  1055→# ⚠️ Orphan Heading Prevention (NOT a page break rule)
  1056→# If an H1 heading would appear in the bottom 15% of the page with no room
  1057→# for at least a few lines of content, push it to the next page.
  1058→# This is NOT "every H1 starts a new page" — it only triggers when the heading
  1059→# would be orphaned at the very bottom.
  1060→available_height = A4[1] - top_margin - bottom_margin
  1061→H1_ORPHAN_THRESHOLD = available_height * 0.15  # ~100pt ≈ 3.5cm — enough for heading + 2 lines
  1062→
  1063→def add_major_section(text, style):
  1064→    """Add an H1-level heading with orphan prevention (NOT forced page break)."""
  1065→    return [
  1066→        CondPageBreak(H1_ORPHAN_THRESHOLD),  # Only break if heading would be orphaned at page bottom
  1067→        add_heading(text, style, level=0),
  1068→    ]
  1069→
  1070→story.extend(add_major_section("Chapter 1: Introduction", styles['Heading1']))
  1071→# ... content ...
  1072→
  1073→doc.multiBuild(story)  # MUST use multiBuild for TOC
  1074→```
  1075→
  1076→**⚠️ CRITICAL TOC LINK RULES:**
  1077→1. `afterFlowable` MUST pass `key` as the 4th element in the notify tuple - without it, TOC entries have no clickable links
  1078→2. `add_heading` MUST set `bookmark_key` AND embed `<a name="key"/>` in the Paragraph text - this creates the link destination
  1079→3. The key must be unique per heading - use a hash of the heading text

  1081→### TOC with Leader Dots (Copy-Paste Ready)
  1082→
  1083→**⚠️ WARNING**: This manual approach creates a visual-only TOC. It has **NO clickable links** and **NO auto-updated page numbers**. Use the auto-generated TOC above (TocDocTemplate + multiBuild) whenever possible. Only use this leader-dots approach if you need very specific visual styling that the auto TOC cannot provide - and even then, prefer the auto approach.
  1084→
  1085→For a professional TOC with leader dots connecting titles to page numbers:
  1086→
  1087→```python
  1088→from reportlab.lib.pagesizes import A4
  1089→from reportlab.platypus import SimpleDocTemplate, Table, TableStyle, Paragraph, PageBreak
  1090→from reportlab.lib.styles import getSampleStyleSheet, ParagraphStyle
  1091→from reportlab.lib import colors
  1092→from reportlab.lib.units import inch
  1093→
  1094→# Setup
  1095→doc = SimpleDocTemplate("report.pdf", pagesize=A4,
  1096→                       leftMargin=0.75*inch, rightMargin=0.75*inch)
  1097→styles = getSampleStyleSheet()
  1098→styles['Heading1'].fontName = 'FreeSerif'
  1099→styles['Heading1'].textColor = colors.black
  1100→
  1101→story = []
  1102→
  1103→# Calculate dimensions
  1104→page_width = A4[0]
  1105→available_width = page_width - 1.5*inch
  1106→page_num_width = 50  # Fixed width for page numbers
  1107→dots_column_width = available_width - 200 - page_num_width
  1108→optimal_dot_count = int(dots_column_width / 4.5)  # ~4.5pt per dot at 7pt font
  1109→
  1110→# Define styles
  1111→toc_style = ParagraphStyle('TOCEntry', parent=styles['Normal'],
  1112→                          fontName='FreeSerif', fontSize=11, leading=16)
  1113→dots_style = ParagraphStyle('LeaderDots', parent=styles['Normal'],
  1114→                           fontName='FreeSerif', fontSize=7, leading=16)
  1115→
  1116→# Build TOC
  1117→toc_data = [
  1118→    [Paragraph('<b>Table of Contents</b>', styles['Heading1']), '', ''],
  1119→    ['', '', ''],
  1120→]
  1121→
  1122→entries = [('Section 1', '5'), ('Section 2', '10')]
  1123→for title, page in entries:
  1124→    toc_data.append([
  1125→        Paragraph(title, toc_style),
  1126→        Paragraph('.' * optimal_dot_count, dots_style),
  1127→        Paragraph(page, toc_style)
  1128→    ])
  1129→
  1130→toc_table = Table(toc_data, colWidths=[None, dots_column_width, page_num_width])
  1131→toc_table.setStyle(TableStyle([
  1132→    ('GRID', (0, 0), (-1, -1), 0, colors.white),
  1133→    ('LINEBELOW', (0, 0), (0, 0), 1.5, colors.black),
  1134→    ('ALIGN', (0, 0), (0, -1), 'LEFT'),
  1135→    ('ALIGN', (1, 0), (1, -1), 'LEFT'),
  1136→    ('ALIGN', (2, 0), (2, -1), 'RIGHT'),
  1137→    ('VALIGN', (0, 0), (-1, -1), 'TOP'),
  1138→    ('LEFTPADDING', (0, 0), (-1, -1), 0),
  1139→    ('RIGHTPADDING', (0, 0), (-1, -1), 0),
  1140→    ('TOPPADDING', (0, 2), (-1, -1), 3),
  1141→    ('BOTTOMPADDING', (0, 2), (-1, -1), 3),
  1142→    ('TEXTCOLOR', (1, 2), (1, -1), TEXT_MUTED),  # From palette --c-muted
  1143→]))
  1144→
  1145→story.append(toc_table)
  1146→story.append(PageBreak())
  1147→doc.build(story)
  1148→```
  1149→
  1150→**MANDATORY Leader Dots Rules:**
  1151→
  1152→| Rule | Requirement |
  1153→|------|-------------|
  1154→| Column widths | ✅ MUST use fixed values. Percentage-based widths are **STRICTLY FORBIDDEN**. |
  1155→| Dot count | ✅ MUST calculate dynamically: `int(dots_column_width / 4.5)`. Hard-coded counts are **STRICTLY FORBIDDEN**. |
  1156→| Page number column | ✅ MUST be at least 40pt wide. |
  1157→| Dot font size | ✅ MUST NOT exceed 8pt. |
  1158→| Dot alignment | ✅ MUST be LEFT-aligned (visual flow from title). |
  1159→| Padding | ✅ MUST be exactly 0 between columns. |

  1178→## Cross-References (Figures, Tables, Bibliography)
  1179→
  1180→Pre-register all figures, tables, and references BEFORE using them in text.
  1181→
  1182→```python
  1183→class CrossReferenceDocument:
  1184→    def __init__(self):
  1185→        self.figures = {}
  1186→        self.tables = {}
  1187→        self.refs = {}
  1188→        self.figure_counter = 0
  1189→        self.table_counter = 0
  1190→        self.ref_counter = 0
  1191→
  1192→    def add_figure(self, name):
  1193→        if name not in self.figures:
  1194→            self.figure_counter += 1
  1195→            self.figures[name] = self.figure_counter
  1196→        return self.figures[name]
  1197→
  1198→    def add_table(self, name):
  1199→        if name not in self.tables:
  1200→            self.table_counter += 1
  1201→            self.tables[name] = self.table_counter
  1202→        return self.tables[name]
  1203→
  1204→    def add_reference(self, name):
  1205→        if name not in self.refs:
  1206→            self.ref_counter += 1
  1207→            self.refs[name] = self.ref_counter
  1208→        return self.refs[name]
  1209→```

  1213→## Image Handling
  1214→
  1215→### Diagram Embedding Rules (MANDATORY)
  1216→
  1217→**Figures are block-level Flowables.** Every image/diagram in ReportLab MUST be appended to `story` as a standalone `Image()` or wrapped in `KeepTogether([image, caption])`. Never simulate floating layouts by placing images inside Paragraph text or multi-column Table cells alongside body text.
  1218→
  1219→**Complex diagrams (>12 nodes)**: Decompose into a ReportLab Table (details) + a simplified diagram image (overview). The diagram gives the big picture; the table carries precision. See SKILL.md "Complex Diagram Strategy".
  1220→
  1221→**Diagram quality**: When generating flowcharts/diagrams for PDF embedding (via Playwright screenshot or ReportLab Drawing), follow the diagram content quality rules in SKILL.md § "Diagram Content Quality Rules". Key points: connectors must not pass through nodes, no high-saturation large fills, multi-arrow convergence must use merge pattern, font sizes must be legible at final embedded size (see context-specific minimums in SKILL.md).
  1222→
  1223→**Cross-brief diagram pipeline**: For flowcharts/architecture diagrams, generate via **Playwright+CSS** (HTML nodes + CSS layout + SVG connectors), screenshot at 2× device scale factor for 300dpi print quality, then embed with `Image()`. See SKILL.md § "Diagram Generation Strategy" for the full pipeline.
  1224→
  1225→**🚫 FORBIDDEN: TikZ standalone → PNG pipeline in Report brief.** Report uses ReportLab, not LaTeX. Going through TikZ requires a LaTeX compiler, adds compilation steps, and models frequently produce broken TikZ code. Use Playwright+CSS instead - it's native to the existing cover rendering engine.
  1226→
  1227→```python
  1228→# Playwright+CSS diagram → PNG → ReportLab embedding
  1229→# 1. LLM generates diagram.html (CSS grid/flexbox nodes + SVG arrows)
  1230→# 2. Playwright screenshots at 2× scale (300dpi equivalent)
  1231→# 3. Embed:
  1232→from reportlab.platypus import Image
  1233→img = Image('diagram.png', width=450)  # auto height via aspect ratio
  1234→story.append(img)                       # block-level Flowable
  1235→```
  1236→
  1237→**⚠️ CRITICAL: Preserve aspect ratio - NEVER hardcode both width and height without reading actual image dimensions.**
  1238→
  1239→**⚠️ CRITICAL: Always constrain to available space - NEVER insert at original size if it exceeds container width/height.**
  1240→
  1241→Pie charts become ellipses, radar charts become diamonds, photos get stretched if you set arbitrary width/height.
  1242→
  1243→**→ MUST READ: `typesetting/overflow.md`** — read the complete bounding box system before writing layout code.
  1244→
  1245→```python
  1246→from PIL import Image as PILImage
  1247→from reportlab.platypus import Image
  1248→from reportlab.lib.pagesizes import A4
  1249→from reportlab.lib.units import inch
  1250→
  1251→def embed_image(path: str, max_width: float = None, max_height: float = None) -> Image:
  1252→    """Embed image with preserved aspect ratio, constrained to container.
  1253→
  1254→    ALWAYS use this pattern. Never hardcode both width and height.
  1255→    If max_width is None, defaults to available_width (page - margins).
  1256→    """
  1257→    if max_width is None:
  1258→        max_width = A4[0] - 2.0 * inch  # Default: A4 width minus 1" margins each side
  1259→    if max_height is None:
  1260→        max_height = A4[1] * 0.35  # ~294pt ≈ 10cm — leaves room for caption + surrounding text on same page
  1261→
  1262→    pil_img = PILImage.open(path)
  1263→    orig_w, orig_h = pil_img.size
  1264→
  1265→    # Scale to fit within BOTH max_width and max_height
  1266→    ratio_w = max_width / orig_w if orig_w > max_width else 1.0
  1267→    ratio_h = max_height / orig_h if orig_h > max_height else 1.0
  1268→    ratio = min(ratio_w, ratio_h)
  1269→
  1270→    return Image(path, width=orig_w * ratio, height=orig_h * ratio)
  1271→
  1272→# Usage:
  1273→available_width = A4[0] - left_margin - right_margin
  1274→img = embed_image('chart.png', max_width=available_width, max_height=300)
  1275→story.append(img)
  1276→```
  1277→
  1278→```python
  1279→# ❌ NEVER do this - distorts the image:
  1280→img = Image('chart.png', width=400, height=250)  # arbitrary height breaks aspect ratio
  1281→```

  1516→#### Option Layout Decision
  1517→```python
  1518→def format_options(options):
  1519→    """
  1520→    Decide option layout based on content length.
  1521→    Short options (≤4 chars each, ≤4 options) → 2×2 grid table
  1522→    Long options or >4 options → vertical list, one per line
  1523→    """
  1524→    max_len = max(len(opt) for opt in options)
  1525→    if len(options) <= 4 and max_len <= 6:  # CJK: 4 chars; Latin: 6 chars
  1526→        # 2×2 grid using Table
  1527→        grid = Table(
  1528→            [[options[0], options[1]], [options[2], options[3]]],
  1529→            colWidths=[200, 200], hAlign='LEFT'
  1530→        )
  1531→        grid.setStyle(TableStyle([
  1532→            ('LEFTPADDING', (0,0), (-1,-1), 24),  # Match option indent
  1533→            ('VALIGN', (0,0), (-1,-1), 'TOP'),
  1534→        ]))
  1535→        return grid
  1536→    else:
  1537→        # Vertical list
  1538→        return [Paragraph(f'{opt}', option_style) for opt in options]
  1539→```

  1581→### Pattern Detection Table
  1582→
  1583→Before writing ANY body paragraph, scan the text. If you find any of these patterns, extract them:
  1584→

  1589→| "Q1→Q2→Q3: 10%→25%→40%" | Inline numbers in text | Chart (matplotlib PNG → `Image()`) |

  1591→| "compared to last year / vs Q2" | Nested comparisons in prose | Side-by-side comparison table |
